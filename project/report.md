# Отчёт по итоговому проекту по курсу «Инженерия Искусственного Интеллекта»

## 1. Паспорт проекта

- **Название проекта:** `Сравнение архитектур CNN для классификации таблеток`
- **Автор:** `Якуничева Ирина Андреевна`
- **Группа:** `ИВБО-08-22`
- **Контакт:** `<https://t.me/Saddfrog`
- **Ссылка на репозиторий:** `<https://github.com/saddfrog/ml_Yakunicheva/tree/main>`

Проект посвящён сравнению производительности трёх популярных архитектур свёрточных нейронных сетей — ResNet50, EfficientNet_B0 и MobileNet_V3 — на задаче классификации 112 типов таблеток. Цель — определить оптимальную архитектуру для реального сервиса распознавания лекарственных средств по фотографии.
---

## 2. Постановка задачи и контекст


1. **Предметная область и задача:**
   - **Пользователь:** Фармацевт или обычный человек, который хочет быстро определить название таблетки по фотографии.
   - **Сценарий:** Пользователь фотографирует таблетку на смартфон, сервис возвращает название и вероятность.
   - **Задача:** Классификация изображений таблеток на 112 классов.

2. **Формулировка в терминах ML**
   - **Входные данные:** Изображение таблетки (RGB, произвольный размер, приводится к 224×224)
   - **Целевая переменная:** Класс таблетки (целое число от 0 до 111)
   - **Ограничения:**
      - Время инференса ≤ 100 мс
      - Возможность запуска на CPU (для мобильных приложений)

3. **Целевые метрики качества:**
   | Метрика | Почему выбрана |
   |---------|----------------|
   | **Accuracy** | Главная метрика качества для сбалансированного датасета |
   | **Weighted F1** | Для учета дисбаланса данных (если есть) |
   | **Precision/Recall** | Для анализа ошибок |
   | **Размер модели** | Важно для деплоя на мобильные устройства |
   | **Время инференса** | Важно для пользовательского опыта |

## 3. Данные


1. **Источник данных:**
   - **Датасет:** OGYEIv2  
   - **Ссылка:** https://www.kaggle.com/datasets/richardradli/ogyeiv2  
   - **Описание:** 112 классов таблеток, 4480 изображений, контролируемые условия съёмки (фиксированный масштаб, однородный фон, два режима освещения).

2. **Структура данных:**
   - Каждый txt-файл содержит первое число — класс таблетки (0-111), остальные числа — координаты полигона (не использовались).

   ogyeiv2:
      train/
         images/ # 3136 изображений
         labels/ # 3136 txt-файлов с метками классов
      test/
         images/ # 672 изображения
         labels/ # 672 txt-файла
      valid/
      images/ # 672 изображения
      labels/ # 672 txt-файла

3. **Предобработка и EDA:**
   - **Шаги предобработки:**
      1. Ресайз изображений до 224×224
      2. Нормализация с mean/std ImageNet
      3. Аугментация: RandomHorizontalFlip (только для train)

   - **EDA (ноутбук `01_eda_ogyeiv2.ipynb`):**
      - Классы сбалансированы (~40 изображений на класс)
      - Изображения имеют разный размер исходно (до 3746×2019)
      - Примеры всех классов визуализированы

## 4. Модели и подходы


1. **Базовые (baseline) модели:**
   - **Baseline:** Модель не использовалась, так как задача — сравнение архитектур.

2. **Улучшенные модели и эксперименты:**
   | Модель | Параметров | Размер | Особенности |
   |--------|------------|--------|-------------|
   | ResNet50 | 25M | ~95 MB | Классическая, глубокая, с skip-connections |
   | EfficientNet_B0 | 5.3M | ~20 MB | Compound scaling, баланс точности и скорости |
   | MobileNet_V3 | 5.5M | ~16 MB | Depthwise convolution, для мобильных устройств |


3. **Нейросетевые модели (если применимо):**
   - **Оптимизатор:** Adam, lr=1e-4
   - **Scheduler:** ReduceLROnPlateau (patience=3, factor=0.5)
   - **Эпохи:** 30
   - **Batch size:** 32
   - **Loss:** CrossEntropyLoss
   **Ноутбук:** `02_train_compare.ipynb`

## 5. Экспериментальный протокол и результаты


1. **Экспериментальный протокол:**
- **Сплит данных:** train/val/test = 70/15/15 (уже разбито в датасете)
- **Метрики считались:** на val каждую эпоху, на test — один раз в конце

2. **Сравнение моделей по метрикам:**

   | Модель | Best Val Acc | Test Acc | Test F1 | Test Precision | Test Recall | Размер |
   |--------|--------------|----------|---------|----------------|-------------|--------|
   | **ResNet50** | 99.70% | **99.85%** | 0.9985 | 0.9987 | 0.9985 | 95 MB |
   | **EfficientNet_B0** | 99.70% | 99.55% | 0.9956 | 0.9965 | 0.9955 | 20 MB |
   | **MobileNet_V3** | 99.26% | 98.51% | 0.9842 | 0.9891 | 0.9851 | 16 MB |

3. **Выбор финальной модели:**
   **Рекомендации в зависимости от сценария:**

   | Сценарий | Рекомендуемая модель | Обоснование |
   |----------|---------------------|-------------|
   | **Максимальная точность** | ResNet50 | 99.85% test accuracy |
   | **Баланс точности и размера** | EfficientNet_B0 | 99.55% при 20 MB (в 4.75 раза меньше ResNet) |
   | **Мобильное приложение** | MobileNet_V3 | 98.51% при 16 MB, самая лёгкая |

## 6. Архитектура решения и сервис


1. **Архитектура пайплайна:**
   - Изображение → Preprocessing (resize 224×224, нормализация) → Модель → Softmax → Класс + Confidence

2. **API и endpoints:**
   | Endpoint | Метод | Описание |
   |----------|-------|----------|
   | `/health` | GET | Проверка работоспособности сервиса |
   | `/predict` | POST | Принимает изображение, возвращает предсказание |
   | `/predict/compare` | POST | Сравнивает все три модели на одном изображении |

   **Пример ответа `/predict`:**
   ```json
   {
   "class_id": 0,
   "class_name": "acc long 600 mg pezsgőtabletta",
   "confidence": 0.9995,
   "top_5": [...],
   "latency_ms": 45.23,
   "model_used": "ResNet50"
   }

3. **Технологический стек:**
   - Python 3.11
   - PyTorch + torchvision
   - FastAPI + Uvicorn
   - Docker
   - scikit-learn, pandas, numpy, matplotlib

## 8. Ограничения и дальнейшая работа

1. **Переобучение модели:** 30 эпох обучения привели к переобучению — лучшие результаты достигнуты на 19-21 эпохах
2. **Небольшой датасет:** 4480 изображений недостаточно для уверенного обобщения
3. **Контролируемые условия съёмки:** датасет снят в лабораторных условиях, на реальных фотографиях качество может быть ниже
4. **Отсутствие early stopping:** модель продолжала обучение после достижения максимума

### Что можно улучшить

| Направление | Предлагаемые улучшения |
|-------------|------------------------|
| **Модели** | Добавить Early Stopping, увеличить аугментации (RandomRotation, ColorJitter) |
| **Данные** | Собрать больше данных (10-20 тыс. изображений) с разным фоном и освещением |
| **Сервис** | Добавить кэширование, batch inference, мониторинг (Prometheus) |
| **MLOps** | Добавить MLflow для трекинга экспериментов |
