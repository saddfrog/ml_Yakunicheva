- **Название проекта:** `Сравнение архитектур CNN для классификации таблеток`

## О проекте

**Тема:** Сравнение производительности архитектур глубокого обучения (ResNet, EfficientNet, MobileNet) при классификации таблеток.

**Датасет:** OGYEIv2 (112 классов, 4480 изображений, контролируемые условия съёмки)

**Цель:** Определить оптимальную архитектуру для задачи распознавания таблеток по критериям:
- точность классификации (Accuracy, F1-score)
- скорость инференса
- размер модели

## 2. Структура проекта

Проект организован в следующей структуре:

- `requirements.txt` – зависимости проекта (библиотеки Python, необходимые для запуска).
- `report.md` – отчёт по проекту (постановка задачи, данные, эксперименты, результаты).
- `self-checklist.md` – чеклист самопроверки проекта перед сдачей.
- `notebooks/` – экспериментальные ноутбуки:
  - EDA, прототипы моделей, отдельные эксперименты.
- `src/` – основной код проекта:
  - модули для работы с данными;
  - обучение моделей;
  - запуск сервиса/скриптов.
- `data/` – демонстрационные/учебные данные:
  - **только открытые, синтетические или обезличенные данные;**
  - при необходимости – небольшие выборки для быстрого запуска.
- `configs/` – конфигурационные файлы:
  - `.yaml`, `.json`, `.env.example` и т.п.
- `tests/` – тесты (юнит-тесты, простые проверки).
- `artifacts/` – сохранённые модели, отчёты, результаты экспериментов.

Если структура проекта отличается от базовой, опишите это здесь.

## 5. Данные

### 5.1. Источник данных

- **Датасет:** OGYEIv2  
- **Ссылка:** https://www.kaggle.com/datasets/richardradli/ogyeiv2  
- **Тип:** Открытый датасет (CC BY 4.0 license)

### 5.2. Структура данных

Датасет содержит 4480 изображений 112 классов таблеток, разбитых на три подвыборки:

| Сплит | Количество изображений | Назначение |
|-------|----------------------|------------|
| **train** | 3136 (70%) | Обучение модели |
| **valid** | 672 (15%) | Валидация во время обучения |
| **test** | 672 (15%) | Финальное тестирование |

**Условия съёмки:**
- Фиксированный масштаб (размер важен для распознавания)
- Однородный фон
- Два режима освещения (верхнее рассеянное и боковое для видимости гравировки)

### 5.3. Хранение данных

В репозитории данные **не хранятся** (добавлены в `.gitignore`).  
Для работы с датасетом необходимо:

1. Скачать датасет с Kaggle:  
   `kaggle datasets download richardradli/ogyeiv2`

2. Распаковать в папку `project/data/raw/ogyeiv2/`

3. Убедиться в структуре:
   ogyeiv2:
      train/
         images/ # 3136 изображений
         labels/ # 3136 txt-файлов с метками классов
      test/
         images/ # 672 изображения
         labels/ # 672 txt-файла
      valid/
          images/ # 672 изображения
          labels/ # 672 txt-файла

Добавить файл extracted_sentences.csv в ogyeiv2 из архивас
### 5.4. Предобработка

- Ресайз изображений до 224×224
- Нормализация с mean/std ImageNet:  
`mean=[0.485, 0.456, 0.406], std=[0.229, 0.224, 0.225]`
- Аугментация: RandomHorizontalFlip (только для train)

**EDA ноутбук:** `notebooks/01_eda_ogyeiv2.ipynb`

## 8. Ограничения и дальнейшая работа

1. **Переобучение модели:** 30 эпох обучения привели к переобучению — лучшие результаты достигнуты на 19-21 эпохах
2. **Небольшой датасет:** 4480 изображений недостаточно для уверенного обобщения
3. **Контролируемые условия съёмки:** датасет снят в лабораторных условиях, на реальных фотографиях качество может быть ниже
4. **Отсутствие early stopping:** модель продолжала обучение после достижения максимума

### Что можно улучшить

| Направление | Предлагаемые улучшения |

| **Модели** | Добавить Early Stopping, увеличить аугментации (RandomRotation, ColorJitter) |

| **Сервис** | Добавить кэширование, batch inference, мониторинг (Prometheus) |
